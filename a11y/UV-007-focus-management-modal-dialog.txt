Description:
Améliorer la gestion du focus dans le composant Dialog (Modal) pour assurer une navigation clavier correcte.

Contexte:
- Dialog.tsx:274 a un disable jsx_a11y/no-noninteractive-tabindex
- Le focus management est crucial pour les utilisateurs clavier/screen reader
- Actuellement le focus peut sortir de la modal

Tâches:

[ ] Gérer le focus initial (premier élément focusable)
[ ] Gérer le retour au focus (élément qui a ouvert la modal)
[ ] Tester avec navigation clavier (Tab, Shift+Tab)
[ ] Gérer la touche Esc pour fermer
[ ] Ajouter des tests e2e de navigation clavier

Implémentation focus trap:
```typescript
const focusableElements = dialogRef.current?.querySelectorAll(
  'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
)
const firstElement = focusableElements?.[0]
const lastElement = focusableElements?.[focusableElements.length - 1]

const handleKeyDown = (e: KeyboardEvent) => {
  if (e.key === 'Tab') {
    if (e.shiftKey && document.activeElement === firstElement) {
      e.preventDefault()
      lastElement?.focus()
    } else if (!e.shiftKey && document.activeElement === lastElement) {
      e.preventDefault()
      firstElement?.focus()
    }
  }
}
```
