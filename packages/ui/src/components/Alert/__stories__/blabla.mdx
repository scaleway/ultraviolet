# Alert - Accessibility

## Summary

- **Status**: ⚠️ Needs Work
- **Type**: Feedback

## Quick Checks

- [⚠️] Keyboard: Close button is keyboard accessible, but no Escape key to dismiss alert
- [⚠️] ARIA: Missing `role="alert"` or `role="status"` for screen reader announcement
- [✅] Focus: Focus visible on close button (inherits from Button component)
- [✅] Contrast: Uses theme colors which should meet WCAG requirements
- [⚠️] Screen Readers: Missing live region attributes, generic close button label

## ARIA Pattern

[ARIA Alert Role](https://www.w3.org/WAI/ARIA/apg/patterns/alert/)

## Rules

Enforced by the component:

- ⚠️ [WCAG 4.1.2: Name, Role, Value - Level A](https://www.w3.org/WAI/WCAG21/quickref/#name-role-value)
- ⚠️ [WCAG 4.1.3: Status Messages - Level AA](https://www.w3.org/WAI/WCAG21/quickref/#status-messages)
- ✅ [WCAG 2.1.1: Keyboard - Level A](https://www.w3.org/WAI/WCAG21/quickref/#keyboard)
- ⚠️ [RGAA 10.2: Dans chaque page web, le contenu visible à la prise de focus est-il cohérent ?](https://accessibilite.numerique.gouv.fr/methode/criteres-et-tests/#10.2)
- ⚠️ [RGAA 10.7: Dans chaque page web, pour chaque élément recevant le focus, la prise de focus est-elle visible ?](https://accessibilite.numerique.gouv.fr/methode/criteres-et-tests/#10.7)

To apply when using the component:

- [WCAG 1.4.1: Use of Color - Level A](https://www.w3.org/WAI/WCAG21/quickref/#use-of-color)
- [RGAA 3.2: Dans chaque page web, le contenu visible porté par les couleurs est-il suffisamment contrasté ?](https://accessibilite.numerique.gouv.fr/methode/criteres-et-tests/#3.2)

## Required Actions

### Critical

1. Add `role="alert"` for important messages or `role="status"` for informational messages
2. Add Escape key handler to dismiss closable alerts
3. Improve close button `aria-label` to include context (e.g., "Close alert" or "Dismiss \{title}")

### Secondary

- Consider adding `aria-live="polite"` or `aria-live="assertive"` based on sentiment
- Add focus management when closing (return focus to previous element)
- Document accessibility props in component JSDoc

## Implementation

```tsx
// Add role and aria-live attributes
export const Alert = ({
  children,
  title,
  sentiment = 'danger',
  // ... other props
}: AlertProps) => {
  const [opened, setOpened] = useState(true)
  const Icon = sentimentIcons[sentiment]

  // Map sentiment to aria-live priority
  const ariaLive = sentiment === 'danger' || sentiment === 'warning' ? 'assertive' : 'polite'

  // Handle Escape key
  const handleKeyDown = (event: KeyboardEvent) => {
    if (event.key === 'Escape' && (closable || onClose)) {
      setOpened(false)
      onClose?.()
    }
  }

  if (!opened) {
    return null
  }

  return (
    <Stack
      role="alert"
      aria-live={ariaLive}
      // ... other props
    >
      {/* ... */}
      {closable || onClose ? (
        <Button
          aria-label={title ? `Close $\{title} alert` : 'Close alert'}
          onKeyDown={handleKeyDown}
          // ...
        >
          <CloseIcon sentiment="neutral" />
        </Button>
      ) : null}
    </Stack>
  )
}
```

## Best Practices

### ✅ Do

```tsx
// Use appropriate sentiment for importance
<Alert sentiment="danger" title="Error" role="alert">
  Your session has expired.
</Alert>

// Provide clear close button context
<Alert closable title="Update available">
  A new version is ready to install.
</Alert>

// Use with action button for important alerts
<Alert
  sentiment="warning"
  title="Unsaved changes"
  buttonText="Save now"
  onClickButton={handleSave}
  closable
>
  You have unsaved changes that will be lost.
</Alert>
```

### ❌ Avoid

```tsx
// Don't use generic close without context
<Alert closable onClose={handleClose}>
  Message here
</Alert>

// Don't use alert for non-urgent messages with assertive role
<Alert sentiment="info" role="alert" aria-live="assertive">
  Tip: You can customize your profile
</Alert>

// Don't remove close functionality without keyboard alternative
<Alert onClose={handleClose}>
  Important message with no way to dismiss via keyboard
</Alert>
```
