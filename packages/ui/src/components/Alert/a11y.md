# Alert - Accessibilité

## Résumé

**Statut** : ⚠️ Needs Work  
**Type** : Feedback/Notification

## Checks rapides

- ✅ Clavier : Navigation OK
- ❌ ARIA : Role manquant (`alert`/`status`)
- ❌ ARIA : `aria-live` manquant
- ⚠️ Focus : Gestion à améliorer
- ✅ Contraste : OK (thèmes light/dark)
- ⚠️ Screen Readers : À tester après fixes ARIA

## Actions requises

### Critiques

1. Ajouter `role="alert"` pour danger/warning
2. Ajouter `role="status"` pour info/success/neutral
3. Ajouter `aria-live="assertive"` (danger/warning) ou `"polite"` (info/success/neutral)
4. Gérer la touche Escape pour fermer les alerts dismissibles

### Secondaires

- Ajouter `aria-labelledby` quand un titre est présent
- Restaurer le focus après dismissal

## Implementation

```tsx
const getAriaAttributes = (sentiment: AlertSentiment) => {
  const isAssertive = sentiment === 'danger' || sentiment === 'warning'
  return {
    role: isAssertive ? 'alert' : 'status',
    'aria-live': isAssertive ? 'assertive' : 'polite',
  }
}
```

## Bonnes pratiques

### ✅ Faire

```tsx
<Alert title="Payment Failed" sentiment="danger" buttonText="Retry">
  Your payment could not be processed.
</Alert>

<Alert sentiment="success" title="Saved" closable onClose={dismiss}>
  Your changes have been saved.
</Alert>
```

### ❌ Éviter

```tsx
// Pas d'auto-dismiss sans contrôle utilisateur
<Alert sentiment="danger" autoDismiss={3000}>
  Critical error
</Alert>

// Pas de multiples alerts en même temps
<>
  <Alert sentiment="error">Error 1</Alert>
  <Alert sentiment="error">Error 2</Alert>
</>
```
