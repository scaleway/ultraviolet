# Alert - Accessibility

## Summary

**Status**: ⚠️ Needs Work  
**Type**: Feedback/Notification

## Quick Checks

- ✅ Keyboard: Navigation OK
- ❌ ARIA: Missing role (`alert`/`status`)
- ❌ ARIA: Missing `aria-live`
- ⚠️ Focus: Management to improve
- ✅ Contrast: OK (light/dark themes)
- ⚠️ Screen Readers: To test after ARIA fixes

## Required Actions

### Critical

1. Add `role="alert"` for danger/warning
2. Add `role="status"` for info/success/neutral
3. Add `aria-live="assertive"` (danger/warning) or `"polite"` (info/success/neutral)
4. Handle Escape key to close dismissible alerts

### Secondary

- Add `aria-labelledby` when a title is present
- Restore focus after dismissal

## Implementation

```tsx
const getAriaAttributes = (sentiment: AlertSentiment) => {
  const isAssertive = sentiment === 'danger' || sentiment === 'warning'
  return {
    role: isAssertive ? 'alert' : 'status',
    'aria-live': isAssertive ? 'assertive' : 'polite',
  }
}
```

## Best Practices

### ✅ Do

```tsx
<Alert title="Payment Failed" sentiment="danger" buttonText="Retry">
  Your payment could not be processed.
</Alert>

<Alert sentiment="success" title="Saved" closable onClose={dismiss}>
  Your changes have been saved.
</Alert>
```

### ❌ Avoid

```tsx
// No auto-dismiss without user control
<Alert sentiment="danger" autoDismiss={3000}>
  Critical error
</Alert>

// No multiple alerts at the same time
<>
  <Alert sentiment="error">Error 1</Alert>
  <Alert sentiment="error">Error 2</Alert>
</>
```
